# 点点小红书创作指南

## 调用方式

### 1:1 模仿模式（默认）

直接按本文档记录的标题、结构、情绪和互动公式创作。调用时至少提供“主题”；若内容涉及个人经历、身份、产品或事件，再提供真实素材。未指定模式时使用本模式。

示例：`用点点风格写一篇“下班后没力气做饭”的小红书，真实情况是我通勤 1 小时。`

默认交付：3 个候选标题、1 篇正文、3—7 个紧贴主题的话题标签。优先保留口语感，不主动补成教程腔或营销腔。

### 融合风格模式

用户明确指出“保留自己的维度”，其余维度套用点点公式。可混合的七个维度为：标题公式、开头模板、正文结构、情感节奏、语言习惯、CTA 策略、发布节奏建议。没有被用户点名保留的维度，默认采用本指南的目标结论；用户没有提供某个自有维度的样例时，先索要样例，不猜测其个人风格。

示例：`用融合风格模式写“第一次去山姆”，保留我自己的语言习惯和结尾方式，其余用点点风格。`

## 证据边界

本指南只依据账号“点点”的 11 篇图文笔记及其中 TOP10 标题、正文摘录和发布统计。样本量小，所得结论是可复用的观察，不是平台普遍规律。

- 已知：昵称“点点”、小红书号 42939260942、简介为空、粉丝数 10、获赞与收藏 10；11 篇均为图文。
- 已知：样本均赞 28.82、均藏 11.45、均评 47.55，互动特征更偏评论驱动；不能把样本均值承诺为未来效果。
- 未知：头像内容、真实职业全貌、长期选题规划、粉丝画像、转化链路、收入和商业合作。不得补写成事实。
- 限制条件：涉及“我”的经历、学校、工作、消费、身体特征、产品效果等，只能使用用户明确提供的真实素材；没有素材时改成提问/观察视角，或先追问。
- 数据中无明确变现线索，不得擅自插入课程、店铺、品牌合作或购买引导。

## IP定位

### 账号搭建示例

以下是基于现有内容的建议，不是对原账号未公开信息的描述：

- 昵称：保留“点点”的亲近感；需要增强识别时可用“点点｜年轻人真实生活问答”。
- 简介：现状为空。可写“记录学生党/打工人的真实困境｜不装懂，评论区一起找答案”。
- 头像：原头像无数据。建议使用清晰真人近景或稳定的单一图形，避免声称原账号采用了某种头像。

### 定位三问结论

1. 面向什么人群：从题材推断，核心可落在大学生、职场新人和低预算年轻人；该人群结论为样本推断，置信度中等。
2. 解决什么问题：把生活中的疲惫、自卑、消费选择、学习/工作困惑变成能让人表达意见、提供经验或获得共鸣的讨论入口。
3. 卖什么产品：当前无证据支持明确产品或服务。创作时先服务互动和人设积累，不设计虚构转化。

### 差异化人设关键词

`不端着`、`敢暴露不完美`、`边经历边求助`、`年轻人真实困境`、`评论区共创答案`。

### 叙事逻辑规则

以“我遇到一个具体问题 → 说出未经美化的感受/限制 → 留下一个别人能回答的缺口”为单篇主线。跨笔记维持“不是专家，是当事人或观察者”的叙事身份；不要忽然切成居高临下的导师口吻。可以跨学生、工作、消费和外貌议题，但每篇都要回到真实的小困境和可讨论的问题。

### 商业模式说明

样本没有店铺带货、知识付费、品牌合作、广告或其他变现证据。默认不写商业植入。若用户另有真实商业目标，先单独确认产品、受众和合规事实，再把 CTA 从“求经验/求判断”改为与目标匹配的动作。

## 运营策略

### 发布节奏建议

样本平均发布间隔为 0.43 天，折合约 10.3 小时；11 篇的小时分布为 06、08、09、10、15、16 和 02 点，其中 10 点 3 篇、15 点 2 篇、02 点 2 篇，其余各 1 篇。这只能证明“发过”，不能证明这些时段效果最好。

- 复刻测试节奏：起步期先每天 1—2 篇，持续 7—14 天，避免直接把短样本的高频更新外推为长期要求。
- 首轮测试窗：08:00—10:00 与 15:00—16:00，因为它们在样本中出现较集中；同题材至少跨两个时段测试后再判断。
- 02:00 只作为已观察时段记录，不因出现 2 次就推荐熬夜发布。
- 每次记录题材、标题公式、发布时间及赞藏评；以评论质量和评论率优先判断互动型内容，不只看点赞。

### 账号阶段判断

按 11 篇、粉丝数 10、简介未搭建且题材跨度较大的现状，判断为起步探索期，置信度高。当前首要目标是找到“哪类真实困境最能稳定引发评论”，而非过早固定商业定位。建议在 20—30 篇后再复盘垂直方向；这是一条运营建议，不是原数据结论。

## 五类标题公式

| 类型 | 可执行公式 | 样本例子 | 使用规则 |
|---|---|---|---|
| 开放求助型 | `具体身份/场景 + 限制条件 + 求经验或清单` | 《借鉴一下你们做外贸的工作图片》；《大学生低预算逛山姆，哪些平价好物值得入》 | 问题要让读者能给出具体答案；正文补足预算、处境或避雷需求。 |
| 争议判断型 | `具体人群/行为 + 是不是/算不算 + 评价词` | 《女生非常省，是加分项吗》 | 保留判断空间，不在正文先宣布唯一正确答案；让评论区站队并解释。 |
| 脆弱暴露型 | `情绪词 + 我的 + 具体不完美` | 《好自卑我的手》；《说出来就好了》可作为较含蓄变体 | 必须用真实细节支撑脆弱，不写空泛的“我破防了”；结尾邀请相似经历。 |
| 现实落差型 | `对象/职业 + 没有想象中这么 + 理想化评价` | 《TikTok主播没有想象中这么好做》；《感觉这样的人真的很有毅力》可用“别人能做到—我做不到”的反差展开 | 正文写期待与现实的具体落差，少做宏大总结。 |
| 事件暗号型 | `圈层事件/专有词 + 那道题/那件事/那个瞬间` | 《四级Disgust那道题》 | 暗号必须对应真实、足够具体的共同事件；正文迅速交代争议点或新信息。 |

## 四原子

每篇至少检查以下四个内容原子；它们可以分布在标题、正文和结尾，不要求机械地各写一段。

1. **钩子（Hook）**：把“人群 + 场景 + 异常/问题”前置。优先直接说“女生非常省”“TikTok 主播”“好自卑我的手”，不用背景铺垫。
2. **情绪锚点（Emotion Anchor）**：落到能感知的口语细节，如“脑子空空身体发软”“6 天了都没有出单”“骨节特别黑”。一次只抓一个主情绪：累、自卑、佩服、焦虑或纠结。
3. **认知缺口（Cognitive Gap）**：留下读者掌握而作者未掌握的信息、评价或经验，例如“哪些值得入”“这是加分项吗”“那道题答案是什么”。不要在正文把问题完全回答掉。
4. **CTA 触发器**：优先用具体低门槛动作收尾，如“评论区直接甩名字”“你们会觉得这是加分项吗”“有同样经历的人吗”。样本中也存在仅靠标题或话题标签触发评论的短正文，因此 CTA 可在标题完成，但不能每篇都堆“点赞收藏关注”。

## 恒定风格

以下为多篇代表作交叉可见的倾向，不声称每篇都同时具备：

- 高频口语入口：`感觉`、`真的`、`你们`、`我`；使用时保持自然，单段不要重复堆砌。
- 人称习惯：当事人题材用第一人称暴露处境，求助或判断题直接转向第二人称/群体“你们”。
- 句式节奏：标题短而直；正文允许连续口语长句交代经历，再用短句落到感受或问题。不要修成整齐排比的公文。
- 情感质地：先承认狼狈、有限预算、不确定或做不到，再求共鸣；不包装成逆袭鸡汤。
- 具体名词：用“四级、Disgust、TikTok、山姆、外卖、美甲”等可感知对象代替抽象概念。
- 标签习惯：样本常用多个紧邻主题的话题标签。生成时控制为 3—7 个，先放场景词、人群词、问题词，不虚构热点标签。
- 签名式收口：开放问题、求清单、求站队或“只有我这样吗”式同类确认；不要统一写“关注我，下期更精彩”。

## 主题适应性

| 主题 | 保留的点点内核 | 调整方式 |
|---|---|---|
| 消费求助 | 低预算、怕踩雷、请别人给清单 | 写清预算/储存/使用人群等约束，结尾要求具体品名或避雷项。 |
| 学习/考试事件 | 圈层暗号、答案不确定 | 标题直接使用真实题目或关键词；正文只补充消息来源和争议点。 |
| 工作体验 | 理想与现实落差、过程疲惫 | 用天数、培训方式、出单情况等真实细节；不凭一个经历概括整个行业。 |
| 外貌/情绪 | 暴露不完美、寻找同类 | 描述具体部位或触发场景，控制自我否定强度，最后问相似经历或实际办法。 |
| 价值判断 | 一句话争议、评论站队 | 让题目可从两面回答；正文不替读者做道德结论。 |
| 日常观察 | 先佩服别人，再照见自己的现实 | 先写可观察行为，再写自身限制，把情绪落在普通生活而非大道理。 |

## 生成流程

1. **收素材**：确认主题、真实身份/经历、必须出现的细节、禁写内容；缺少关键事实就追问。
2. **选模式**：默认 1:1 模仿模式；融合风格模式先列出用户保留的维度和改用点点风格的维度。
3. **定标题**：根据内容意图只选一个主公式，生成 3 个候选；不为追求争议而改写事实。
4. **搭四原子**：用一句话分别确定 Hook、Emotion Anchor、Cognitive Gap、CTA，再合并成自然正文。
5. **调风格**：删掉教程腔、营销腔、空泛升华和过度工整排比；保留必要的口语停顿与具体名词。
6. **配标签**：从真实题材抽取 3—7 个场景、人群和问题标签；不声称使用了未经验证的热词。
7. **按默认格式交付**：依次输出“候选标题 / 正文 / 话题标签 / 建议发布测试窗”。融合模式额外用一行说明维度选择。

## 发布前检查

- [ ] 标题是否属于五类公式之一，且没有空标题？
- [ ] 开头是否在第一句给出具体场景、情绪或圈层暗号？
- [ ] 正文是否包含一个可感知的真实细节，而不是只有抽象情绪？
- [ ] 四原子是否齐全，认知缺口是否仍允许读者作答？
- [ ] 全篇是否只维持一个主情绪，且没有硬升华为鸡汤？
- [ ] CTA 是否具体、低门槛，并避免机械的“点赞收藏关注”？
- [ ] 个人经历、职业、消费、身体、产品和商业信息是否都来自用户提供的事实？
- [ ] 是否把建议、样本观察与已知事实分开表达，没有把 11 篇小样本外推成平台规律？
- [ ] 融合模式是否严格保留用户指定维度，其余维度才套用点点结论？
- [ ] 标签是否为 3—7 个且紧贴正文；发布时间是否被标注为测试建议而非效果保证？

# 点点小红书账号蒸馏产出设计

## 目标

基于 TikHub 采集到的点点账号 11 篇公开笔记，生成两个最终产物：

1. `skills/dl-xhs-distill/output/点点_创作指南.skill/SKILL.md`
2. `skills/dl-xhs-distill/output/点点_蒸馏报告.html`

用户选择“强模仿型”口径。创作指南优先复刻当前样本中可观察到的标题、口语表达、情绪与互动机制；分析报告仍保持证据边界，不把早期样本包装成成熟稳定的 IP。

## 证据边界

- 样本量为 11 篇，全部为图文，发布时间集中，账号粉丝数为 10。
- 只把跨笔记重复出现或数据明显支持的特征写成稳定规律。
- 对昵称、简介、头像、商业模式等缺乏信息的项目明确标注“未建立”或“无法从当前数据确认”。
- 不把单篇高互动笔记直接等同于稳定爆款公式。
- 报告中的“模仿建议”与“事实判断”分开表达。

## 创作 Skill 设计

- 默认调用方式是 1:1 强模仿；同时保留融合风格模式。
- 强模仿重点包括：短标题、口语化自述、生活困境或脆弱感、争议问句、低门槛评论请求、标签补充语境。
- 至少提供七个可混合维度：标题公式、开头模板、正文结构、情感节奏、语言习惯、CTA 策略、发布节奏建议。
- 给出五类标题公式及原创示例，避免机械复刻原笔记文本。
- 将账号主题分散、正文信息密度不稳定、依赖评论区等弱点列入使用限制，但不降低默认模仿强度。

## HTML 报告设计

- 三层主体结构：IP定位、运营策略、内容形式。
- 对 TOP10 笔记逐条人工归入五类标题公式，并解释分类依据。
- 代表作精读覆盖：外贸求图、山姆求推荐、消费观争议、个人自卑、TikTok 工作经历。
- 区分跨笔记恒定特征与主题适应性特征。
- 商业模式结论为“暂未形成可确认的变现结构”。
- 视觉严格遵循蒸馏任务：`#FAFAF9` 背景、`#5546FF` 唯一强调色、无衬线字体、16px 大圆角白卡、轻阴影、桌面端侧边蒸馏进度轨、移动端顶部进度条。
- 使用单文件 HTML、手写 CSS、原生 JavaScript、`details/summary` 折叠面板，不引入外部框架。

## 验证

- 运行 `scripts/utils/quality.py` 的 `check_outputs` 校验。
- 确认 HTML 包含“IP定位”“运营策略”“内容形式”三个锚点和实质内容。
- 确认创作指南是 `.skill` 文件夹内的非空 `SKILL.md`。
- 在浏览器或渲染工具中检查桌面端与 768px 以下移动端布局、进度轨、折叠面板和首屏动画。
- 最终回报文件绝对路径、核心判断及验证结果。
